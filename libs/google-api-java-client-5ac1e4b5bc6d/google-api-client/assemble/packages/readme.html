Packages for the ${project.name} ${project.version}

While developing a Java application for mobile devices (such as on Android), it is convenient to use the ${project.artifactId}-${project.version}.jar from the root directory. Unfortunately, its large size makes it unwieldy, especially since you are likely to only use a relatively small subset of the library. That results in a slower download of your application, and a slower start time for your application as it is loaded into memory. If multiple applications on the end-user's mobile device include a copy of the whole, then the problem grows worse for the end user for each new application.

One possible solution is to use the packaged jars in this directory instead. There is exactly one jar for each Java package in the client library, and you only include what you need. Be careful to understand the acyclic package dependency structure, as is specified in the JavaDoc. It is more difficult to set up once, but the end goal is that you release a smaller application, and your end users will (hopefully) appreciate it.

Another possible alternative solution that has been proposed is to try to use ProGuard. We do not have any experience with it.
